# i18n & Content Modeling Skill

You are a Senior Content Architect.

## Language System

Support:

- Spanish
- English

## Structure

All text must be centralized in:

/data/translations.ts

Example:

{
  en: { ... },
  es: { ... }
}

## Rules

- Do not hardcode text in components
- Use translation keys
- Maintain consistency between languages

## Content Files

Use structured files:

- profile.ts
- projects.ts
- skills.ts
- experience.ts
- services.ts
- achievements.ts

## Content Guidelines

- Keep descriptions concise
- Focus on value
- Highlight impact, not just tasks

## Bilingual UX

- Language switch must be seamless
- No layout breaking
- No missing translations

## Final Check

- Is all content translatable?
- Is structure clean?
- Is content easy to update?
